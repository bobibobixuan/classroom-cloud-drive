# 🚀 课堂互动云盘 (Classroom Cloud Drive)

这是一个专为“纯离线/严格限制的计算机机房”设计的极简、高效、免配置的作业收集与课堂互动系统。
采用 `Go + Python` 双子星架构，所有前后端环境均打包为独立 `.exe` 文件，**零环境依赖，U盘即插即用**。

## ✨ 核心特性

- 📦 **极简部署**：无需安装 Python、Go 或任何 Web 服务器（如 Nginx/Apache），双击即可运行。
- 🔐 **安全注册**：支持使用教师手机号作为统一“邀请码”进行注册，防止机房外人员恶意连接。
- 💾 **容量配额**：内置单人 500MB 存储限额机制，防止恶意大文件塞满教师机硬盘。
- 💬 **互动聊天室**：内置全班实时聊天板，支持一键发送“文件提取码”，学生间无缝共享示例代码与素材。
- ⚡ **内网极速**：完全基于局域网传输，千兆机房环境下上传下载瞬间完成，不消耗公网带宽。

---

## 🏗️ 技术架构

- **前端 (UI)**：原生 HTML/CSS/JS 构建，无任何外部 CDN 依赖，纯离线可用。
- **代理节点 (Go)**：使用 Go 语言编写反向代理与静态文件服务，利用 `//go:embed` 将前端页面直接内嵌至 `.exe`，同时负责无视 HTTPS 证书阻拦进行流量打通。
- **业务后端 (Python)**：基于 `FastAPI` 构建，使用 `SQLite` 单文件数据库，通过 `PyInstaller` 封装为独立服务端。

## 🎒 机房发车指南（部署教程）

### 第一步：大本营准备（运行服务端）
1. 将 `云盘服务端.exe` 放置在教师机（或大本营服务器）上。
2. 双击运行。程序会在同级目录自动生成 `classroom.db` (数据库) 和 `classroom_data/` (学生作业存储夹)。
3. **关键操作**：Windows 防火墙可能会拦截该程序，请务必在“允许应用通过 Windows 防火墙”中，将其对应的【专用】和【公用】网络权限打勾。

### 第二步：启动前台代理（机房接入）
1. 将 `课堂云盘系统.exe` 拷贝至 U 盘带到机房。
2. 在机房教师机上，**右键 -> 以管理员身份运行** `课堂云盘系统.exe`。
3. 若弹出防火墙警告，请允许访问。

### 第三步：学生访问
查看代理黑框中打印的地址，将地址写在黑板上，学生在浏览器输入即可访问。

## ⚠️ 避坑指南：关于“本地域名 (.local)”无法访问的说明

在最初的设计中，系统通过 mDNS（多播 DNS）广播了 `http://pan.local` 的优美域名。但在**真实的学校机房环境**中，该域名大概率无法解析，表现为网页打不开。

### 🔍 失败原因：
1. **AP 隔离 / 多播拦截**：机房的交换机为了防止网络风暴，通常在物理层面上禁用了局域网组播/广播数据包。
2. **虚拟网卡干扰**：如果教师机安装了虚拟机（VMware 等），mDNS 广播可能会错误地发送到虚拟局域网中，导致真实物理机房的学生收不到。
3. **Windows 防火墙限制**：Windows 默认在“公用网络”下会严格拦截局域网域名解析。

### 💡 终极替代方案（机房实战推荐）：

在断网机房中，抛弃花里胡哨的域名，直接使用以下两种最底层的寻址方式：

- **方案 A：直接使用 IP 地址（绝对防弹，强烈推荐）**
  查看 Go 代理黑框中打印的 `http://192.168.x.x`。这是最底层的物理协议，只要插了网线，无视任何交换机拦截，**100% 连通**。

- **方案 B：使用计算机名（NetBIOS）**
  在浏览器直接输入教师机的计算机名（例如：`http://Teacher-PC`）。在没有 DNS 服务器的纯内网中，Windows 自带的 NetBIOS 协议可以实现相互解析。
